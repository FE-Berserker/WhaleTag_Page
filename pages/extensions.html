Manifest 与调度

每个扩展在构建后生成 manifest.json,包含以下字段:

  • id、name、type(viewer/editor)、color、fileTypes、entryPoint、enabled、isDefault。

双击文件或选择「Open With」时的调度顺序为:

  1. 用户为该扩展设置的默认关联。
  2. 扩展 manifest 中的 isDefault 标记。
  3. 任意匹配该扩展名的内置扩展。
  4. 回退到系统默认应用。

双层 iframe 拓扑

excalidraw-editor 与 drawio-editor 这类第三方 webapp 编辑器采用双层 iframe:

  • 外层: WhaleTag 的 ExtensionHost iframe,负责与主进程 postMessage 通信。
  • 内层: 第三方 webapp 本身运行在另一个 iframe 中,通过外层桥与主进程交互。

要让这种拓扑正常工作,必须同时满足四件套:

  1. 在 app.ready 之前注册特权协议 wale-extension://(standard + secure)。
  2. 主进程不对 whale-extension:// 响应套 CSP,由每个扩展的 HTML meta CSP 自行治理。
  3. 构建第三方 webapp 时不能过滤子目录,确保子资源 200。
  4. Draw.io 通过 ?proto=json 参数切换到结构化 JSON 协议。

修订历史

编辑类扩展在保存前会自动创建修订备份,保存在 .whale/revisions/<basename>/<ts>.<ext>,可随时回溯。

  • IPC 端点: ext:backupRevision、ext:writeFile、ext:listRevisions、ext:restoreRevision。
  • 清理: 超过 30 天的旧修订会被自动清理。
  • UI: 通过 RevisionHistoryDialog 查看历史版本并一键恢复。

安全模型

  • 扩展运行在 whale-extension:// 特权协议的沙箱 iframe 中。
  • 扩展与主进程通过 postMessage 通信,主进程严格校验 event.source === iframe.contentWindow。
  • 每个扩展 HTML 自带独立的严格 CSP meta 标签,防止内嵌脚本逃逸。
  • 扩展 iframe 的 sandbox 属性包含 allow-same-origin allow-scripts allow-modals allow-downloads。
