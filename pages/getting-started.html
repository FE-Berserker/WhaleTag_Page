快速开始

从源码克隆到本地,启动 WhaleTag 并开始管理你的第一个文件夹位置。

环境要求

  • Node.js ≥ 18
  • npm(随 Node.js 一起安装)
  • Git(用于克隆仓库)

安装与启动

  1. 克隆仓库并进入目录:
    git clone git@github.com:FE-Berserker/WhaleTag.git && cd WhaleTag
  2. 安装依赖(会下载 Electron 二进制,首次较慢):
    npm install
  3. 启动开发模式:
    npm run dev
注意: 启动前请确保 shell 中 ELECTRON_RUN_AS_NODE 已被 unset。残留值会让 Electron 退化成纯 Node 进程,主进程窗口无法弹出。

添加第一个位置

启动后,点击左侧位置面板中的「添加位置」,选择一个本地文件夹。WhaleTag 会在该文件夹下创建 .whale/ 元数据目录,用于存储标签、索引和缩略图,但不会修改原文件名或文件夹结构。

常用脚本

脚本作用
npm run dev开发模式:webpack serve + electronmon 热重载
npm run build生产构建,输出到 release/app/dist

npm run lintESLint 代码检查
npm run type-checktsc --noEmit 类型检查
npm testElectron 测试运行器,覆盖 70+ .test.ts 文件
npm run clean清理构建输出

构建与打包

生产构建会编译主进程、渲染层与 15 个扩展,输出到 release/app/dist。打包则通过 electron-builder 生成安装程序。

# 生产构建
npm run build

# 运行生产版本
npm start

# 打包当前平台
npm run package

# 打包指定平台
npm run package:win   # Windows NSIS
npm run package:mac   # macOS DMG
npm run package:linux # Linux AppImage
离线打包提示: 如果处于受限网络,可设置 ELECTRON_BUILDER_NSIS_RESOURCES_DIR 指向本地 nsis-resources 目录,避免从 GitHub 下载。详见 docs/14-packaging.md。

打包前请确保:

  • ELECTRON_RUN_AS_NODE 已 unset。
  • 已关闭正在运行的 WhaleTag 进程,避免文件占用导致打包失败。
  • 原生依赖(better-sqlite3、sharp 等)已通过 @electron/rebuild 完成 Electron ABI 重建。

可选运行时工具

以下工具不是 WhaleTag 运行时必需,但会扩展特定格式的支持能力。请自行下载并放到 tools/ 目录(这些工具体积较大,不进 git)。

  • LibreOffice: 用于 Office 文件缩略图与 office-viewer 预览(doc/docx/xls/xlsx/ppt/pptx/odt/ods/odp)。
  • ODAFileConverter: 用于 DWG 等 CAD 文件转换与缩略图。

故障排查

  • 启动时报 registerSchemesAsPrivileged / app is undefined: 检查 ELECTRON_RUN_AS_NODE 是否已 unset,而不是仅设为空字符串。
  • 打包报文件占用或 EBUSY: 先结束所有 WhaleTag.exe / electron.exe 进程,再执行 npm run package:win。
  • 首次添加大目录后搜索/缩略图加载慢: 这是正常的首次索引过程;后续增量更新会快很多。
  • Windows 任务栏图标仍是 Electron 默认鲸鱼: 当前离线打包配置跳过了 winCodeSign 图标嵌入,是已知问题。
  • AI 助手提示 API key 无效: 旧安装迁移到新机器/用户后,DPAPI 加密的 key 会失效,需要重新设置。
