AI 助手

WhaleTag 内置 AI 助手,可通过 Claude Code CLI 或兼容的 HTTP 服务提供流式对话。

Provider 配置

AI 助手支持三种 provider,可在设置面板切换:

  • Claude Code CLI: 内置 bundled 的 Claude Code CLI,通过 @anthropic-ai/claude-agent-sdk 调用。注意:只是免去安装步骤,仍需要 Anthropic 鉴权(API key 或 ~/.claude 凭证)。
  • Ollama: 连接本地 Ollama 服务(如 http://localhost:11434),适合离线使用。
  • OpenAI / 兼容端点: 任何 OpenAI 兼容 API,API key 通过 Electron safeStorage 加密存储。

Ollama 与 OpenAI 共享同一个 HttpChatProvider 运行时。

IPC 通道

主进程与 AI 侧栏之间共有 16 个 IPC 通道:

3 个推送通道

  • ai:chunk — 流式内容片段
  • ai:error — 错误信息
  • ai:approvalRequest — 需要用户批准的工具调用

13 个 invoke 通道

通道用途
ai:query发送用户消息并开始对话
ai:cancel取消当前生成
ai:prewarm预热 Claude CLI 进程池
ai:generateTitle自动生成会话标题
ai:inlineEdit内联编辑选中文本
ai:setApiKey / ai:clearApiKey / ai:hasApiKey管理 Anthropic API key
ai:setOpenaiKey / ai:clearOpenaiKey / ai:hasOpenaiKey管理 OpenAI API key
ai:discoverCli探测可用的 Claude CLI 路径
ai:resolveApproval用户批准或拒绝工具调用

工具能力

HTTP provider 向模型暴露 5 个工具,Claude CLI 也可通过类似机制操作文件:

  • read_file — 读取文件内容
  • list_directory — 列出目录
  • list_tags — 获取当前位置的标签与描述
  • write_file — 写入文件(需批准)
  • apply_tag — 为文件打标签,自动处理互斥家族(需批准)

所有非只读工具都会经过统一的 decideToolCall 闸门,单次对话最多允许 12 轮工具调用。

只读护栏

AI 助手对文件系统的所有写操作都需要经过统一的 decideToolCall 闸门,默认采用只读策略,避免 AI 误改文件。

  • readOnlyGuard 会拒绝写工具命中只读根,也会拒绝在只读目录下执行 Bash 工具。
  • ANTHROPIC_API_KEY / OPENAI_API_KEY 使用 Electron safeStorage 加密保存,不会以明文形式暴露在配置文件或 localStorage 中。
  • decideToolCall 对 Claude CLI 与 HTTP provider 均生效,是统一的安全闸门。

Windows 注意事项

  • .cmd shim 问题: Windows 上通过 .cmd shim 启动 Claude CLI 可能破坏 SDK 的 stdio,优先使用原生 .exe 或 Node 入口。
  • asar 路径重映射: 主进程传给外部 Node 进程的路径需要从 app.asar 映射到 app.asar.unpacked,否则会出现 MODULE_NOT_FOUND。
  • 仍需鉴权: bundled Claude Code 只是内置了二进制,仍需 Anthropic 鉴权才能使用;可优先使用本地 Ollama 绕过。
