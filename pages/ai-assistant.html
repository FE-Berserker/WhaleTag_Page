AI 助手

WhaleTag 内置 AI 助手,可通过 Claude Code CLI 或兼容的 HTTP 服务提供流式对话。

两种运行模式

  • Claude Code CLI 嵌入: 直接调用本地的 Claude Code CLI,适合已经安装并配置好 Claude Code 的用户。
  • HTTP Provider: 兼容 ollama 与 OpenAI 格式,可连接本地模型或自定义 API 端点。

通信通道

  • 3 个 IPC 推送: ai:chunk(流式内容)、ai:error(错误)、ai:approvalRequest(需要用户批准)。
  • 13 个 invoke 通道: 用于发送消息、切换 provider、管理会话等。

只读护栏

AI 助手对文件系统的所有写操作都需要经过统一的 decideToolCall 闸门,默认采用只读策略,避免 AI 误改文件。

密钥存储

API key 使用 Electron 的 safeStorage 加密保存,不会以明文形式暴露在配置文件或 localStorage 中。

提示: 当前开发分支可能在 Windows 上遇到 Claude Code CLI 二进制启动失败的问题(libc 不匹配警告)。不影响 WhaleTag 主体功能,仅影响 AI 侧栏;可优先使用 HTTP Provider 或等待后续修复。
