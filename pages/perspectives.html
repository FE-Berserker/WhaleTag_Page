视角

用不同的方式查看同一批文件,从 9 种视角中选择最适合当前任务的那一种。

目录
基础浏览视图 Task 三档子视图 日历视图 关系与聚合视图 全局 viewDepth 拖拽打标签 旧视图迁移

基础浏览视图

  • List: 列表视图,文件信息最完整,适合批量查看元数据。
  • Grid: 网格视图,缩略图与文件名并排,平衡信息密度。
  • Gallery: 画廊视图,突出媒体预览,适合图片/视频浏览。

Task 三档子视图

task 并不是一个单一视图,而是一个「第三档」视图:选中 task 后,可在 Kanban / Matrix / Gantt 三个子视图间切换。子视图状态保存在全局 localStorage.whale-task-subview 中,对所有文件夹一致。

  • Kanban: 按工作流阶段分列,支持拖拽卡片、period 标签拖放。
  • Matrix: 艾森豪威尔 2×2 矩阵(重要 × 紧急),按象限标签布局,未打标签文件进入独立托盘。
  • Gantt: 纯 DOM 实现,无 ECharts。支持横向滚动、useBarDrag 拖拽调整起止、键盘导航、按工作流分泳道、快捷范围预设(1w/2w/1m/1q)、PNG 导出,以及逾期/进行中视觉标记。

日历视图

  • Calendar: 5 档粒度(日 / 周 / 月 / 年 / 日程),按智能日期或期间标签渲染事件。
  • Mapique: 地图视图,基于文件的 geo:lat,lng 标签展示位置,默认使用高德瓦片。

关系与聚合视图

  • Folderviz: 文件夹可视化,支持树状图、径向图、矩形树图、旭日图,有独立的隐藏/过滤状态。
  • Tag Cloud: 标签云,字体大小按文件数量平方根缩放。
  • Knowledge Graph: 基于 xyflow 的文件–标签二分图可视化。

全局 viewDepth

  • 全局递归深度 viewDepth 范围 [1, 5],默认 1,在工具栏可快速调整。
  • 所有视角(除 FolderViz 外)共享 DirectoryContentContextProvider 的同一数据集。
  • 深度变化有 200ms 防抖;递归加载上限 MAX_RECURSIVE_ENTRIES = 10000。
  • Tag Cloud 与 Knowledge Graph 中排序控件会被隐藏,因为它们是「无效控件」。

拖拽打标签

任意视角都支持从左侧标签库拖拽标签 chip 到条目上打标签:

  • 支持多选批量打标签。
  • 支持拖拽到文件夹、卡片、磁贴、地图标记、图谱节点。
  • 拖拽 period: 标签到文件会弹出 PeriodTagDialog 选择日期范围。

旧视图迁移

早期版本使用了一些旧视图名,加载文件夹时会通过 migrateViewMode 自动映射:

  • mindmap → knowledge-graph
  • kanban / matrix → task
